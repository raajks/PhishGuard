ABSTRACT
================================================================================

Phishing attacks have become a major cybersecurity threat, leveraging social 
engineering techniques to deceive users into disclosing sensitive information. 
With the rapid growth of online services, phishing strategies are continuously 
evolving, making their detection increasingly challenging.

This paper presents a comprehensive survey of phishing detection techniques, 
covering the progression from traditional approaches to advanced machine 
learning and deep learning-based methods. Existing techniques are broadly 
categorized into four groups: list-based methods, heuristic and similarity-based 
methods, machine learning-based approaches, and deep learning-based models. 
For each category, the paper discusses fundamental concepts, commonly used 
features and datasets, as well as their advantages and limitations.

Furthermore, a comparative analysis of these techniques is provided, 
highlighting key challenges such as zero-day attack detection, data imbalance, 
feature selection, and model generalization. The study also identifies future 
research directions aimed at enhancing the accuracy, scalability, and 
adaptability of phishing detection systems.

This survey aims to serve as a structured and up-to-date reference for 
researchers and practitioners in the field of cybersecurity.

2. LITERATURE REVIEW
================================================================================

2.1 URL-Based Detection Methods
    - Garera et al. (2007) - Logistic Regression on URL features
    - Ma et al. (2009) - Online learning for malicious URL detection
    - Le et al. (2011) - URL lexical features

2.2 Content-Based Detection Methods
    - Zhang et al. (2007) - CANTINA system using TF-IDF
    - Xiang et al. (2011) - CANTINA+ with HTML/DOM features
    - Visual similarity-based approaches

2.3 Machine Learning Approaches
    - Abdelhamid et al. (2014) - Multi-label classification
    - Mohammad et al. (2014) - UCI Phishing Websites dataset (used in this work)
    - Sahingoz et al. (2019) - NLP-based URL features + 7 ML classifiers
    - Rao & Pais (2020) - CatchPhish with URL-based features

2.4 Deep Learning Approaches
    - Bahnsen et al. (2017) - LSTM for URL classification
    - Yang et al. (2019) - CNN for phishing detection
    - Wei et al. (2020) - Character-level CNN
    - Aljofey et al. (2022) - Bi-LSTM with attention mechanism

2.5 Hybrid and Ensemble Methods
    - Combination of ML + heuristic rules
    - Random Forest ensembles (used in PhishGuard)
    - Voting classifiers and stacking

2.6 Research Gap
    - Most systems rely on only ML or only rules
    - Few combine both with smart override mechanisms
    - Need for interpretable results (not just Safe/Phishing label)

3. PHISHING DETECTION TECHNIQUES — TAXONOMY
================================================================================

3.1 Feature Categories

    3.1.1 URL-Based Features
          - URL length, special characters (@, --, //)
          - IP address in URL
          - URL shortening services
          - Suspicious TLDs (.xyz, .tk, .ml, .cf)
          - Number of subdomains
          - HTTPS presence

    3.1.2 Domain-Based Features
          - Domain age and registration length
          - WHOIS information
          - DNS records
          - SSL certificate validity

    3.1.3 Content-Based Features
          - HTML/JavaScript analysis
          - Form actions and redirections
          - iFrame usage
          - Right-click disabling
          - Pop-up windows
          - Status bar customization

    3.1.4 External-Based Features
          - Google Safe Browsing index
          - PageRank
          - Alexa ranking
          - VirusTotal scan results

3.2 Classification Algorithms Comparison

    Algorithm               | Accuracy | Pros                    | Cons
    ----------------------- | -------- | ----------------------- | ----------------------
    Logistic Regression     | ~89%     | Fast, interpretable     | Limited non-linearity
    Decision Tree           | ~91%     | Interpretable           | Overfitting risk
    Random Forest           | ~95%     | High accuracy, robust   | Less interpretable
    SVM                     | ~93%     | Good generalization     | Slow on large data
    Naive Bayes             | ~87%     | Very fast               | Feature independence
    Neural Network (MLP)    | ~94%     | Non-linear patterns     | Black box
    LSTM                    | ~96%     | Sequential URL patterns | Needs more data
    CNN                     | ~95%     | Character-level features| Complex architecture

3.3 Why Random Forest Was Chosen
    - Best balance of accuracy (~95%) and interpretability
    - Feature importance ranking available
    - Resistant to overfitting with proper hyperparameters
    - Fast inference for real-time web application

4. PROPOSED SYSTEM — PhishGuard
================================================================================

4.1 System Architecture

    URL Input → Validation → Feature Extraction (30 features)
                                  ↓
                  ┌───────────────┴───────────────┐
                  │                               │
           ML Prediction (70%)          Rule-based Score (30%)
        Random Forest Classifier      Keyword, TLD, IP, HTTPS checks
                  │                               │
                  └───────────────┬───────────────┘
                                  ↓
                         Hybrid Score (0-100)
                                  ↓
                         Override Rules Applied
                                  ↓
                   Safe | Suspicious | Phishing

4.2 Feature Extraction Module (30 UCI Features)
    - Based on Mohammad et al. (2014) UCI dataset features
    - Features encoded as: -1 (Phishing), 0 (Suspicious), 1 (Legitimate)
    - Categories: URL-based (12), Domain-based (6), HTML-based (7), External (5)

    Complete Feature List:
    1.  Having IP Address          16. Server Form Handler (SFH)
    2.  URL Length                  17. Abnormal URL
    3.  Shortening Service         18. Website Forwarding
    4.  Having @ Symbol            19. Status Bar Customization
    5.  Double Slash Redirecting   20. Disabling Right Click
    6.  Prefix/Suffix (-)         21. Using Pop-up Window
    7.  Having Sub-domain          22. IFrame Redirection
    8.  SSL/HTTPS                  23. Domain Mismatch
    9.  Domain Reg. Length         24. Fake Favicon
    10. Favicon                    25. Domain In Title
    11. Non-Standard Port          26. WHOIS Known By
    12. HTTPS in Domain URL        27. Google Index
    13. Request URL                28. Links In Comments
    14. URL of Anchor              29. SFH Check
    15. Links In Script Tags       30. Abnormal URL Flag

4.3 ML Model — Random Forest Classifier
    - Algorithm: Random Forest (Ensemble of Decision Trees)
    - Estimators: 100 trees
    - Max Depth: 15
    - Min Samples Split: 5
    - Min Samples Leaf: 2
    - Training Data: UCI Phishing Websites (11,055 samples)
    - Train/Test Split: 80/20 with stratification

4.4 Rule-Based Engine
    - 50+ whitelisted safe domains (Google, GitHub, Facebook, etc.)
    - Risk score calculation (0–100):
      * IP address in URL:       +30 points
      * Missing HTTPS:           +20 points
      * Suspicious keywords:     +20 points
      * URL length > 75 chars:   +10 points
      * Dash in domain:          +10 points
      * Suspicious TLD:          +10 points
      * @ symbol present:        +15 points
      * URL shortener detected:  +10 points
      * Multiple subdomains:     +10 points

4.5 Hybrid Scoring Formula
    - Formula: Hybrid Score = (ML Score × 0.7) + (Rule Score × 0.3)
    - Thresholds: Safe (<20), Suspicious (20-50), Phishing (>50)

4.6 Override Rules (Smart Corrections)
    - Known safe domain detected → Force "Safe" label
    - IP address as URL → Force "Suspicious" label
    - No HTTPS + login keyword → Force "Phishing" label

4.7 Additional Analysis Modules
    - Domain Information: TLD, subdomain, IP resolution
    - Brand Detection: Recognition of 15+ major brands
    - SSL Certificate: Validity, issuer, trust status
    - Hosting Provider: IP prefix matching (Google, AWS, Cloudflare, etc.)
    - Geolocation: Country/city from IP prefix
    - Threat Intelligence: Phishing history, malware, spam reports
    - Human-readable Explanations: Feature-based reasoning with severity levels

4.8 Web Application
    - Backend: Flask (Python)
    - Frontend: HTML5, CSS3, JavaScript, Bootstrap 5
    - Design: Dark glassmorphism with animated radar scanner
    - API: REST endpoint (POST /predict) for integration
    - Deployment: Render cloud platform

6.2 Advantages of PhishGuard
    - Hybrid approach reduces false positives
    - Override rules handle edge cases (whitelisted domains, IP URLs)
    - Rich output: domain info, SSL, hosting, geolocation, threat intel
    - Human-readable explanations with severity levels
    - REST API for integration with other tools
    - Modern web interface with real-time scanning

7. CONCLUSION & FUTURE WORK
================================================================================

7.1 Conclusion
    - Surveyed phishing detection from traditional blacklisting to deep learning
    - Proposed PhishGuard: a hybrid ML + Rule-based system
    - Achieved ~95% accuracy on UCI dataset with real-time web interface
    - Demonstrated that combining ML with domain-specific rules improves robustness
    - Provided interpretable results unlike black-box models

7.2 Future Work
    - Integrate real-time WHOIS lookup and DNS analysis
    - Add content-based features (HTML scraping)
    - Implement deep learning models (LSTM, Transformer) for comparison
    - Use live phishing feeds (PhishTank, OpenPhish) for continuous learning
    - Add browser extension for real-time protection
    - Implement user feedback loop for model improvement
    - Scale with distributed architecture for enterprise use
